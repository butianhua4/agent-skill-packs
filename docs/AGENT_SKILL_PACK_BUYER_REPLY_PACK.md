# Agent Skill Pack Buyer Reply Pack

Use this pack when a buyer asks about custom AI agent skills, workflow prompts,
agent rules, or reusable automation playbooks.

## Fast Fit Check

Use this before quoting:

```text
Thanks for reaching out. This is a good fit if you need a reusable AI agent
skill pack, prompt workflow, safety rule set, test cases, or delivery template.

To scope it safely, please send:
1. The workflow you want the agent to support
2. The platform or tool it will run in
3. One redacted sample input
4. The expected output format
5. Anything the agent must never do

Please do not send passwords, tokens, private keys, payment details, ID
documents, or private customer data.
```

## First Reply For A Vague Inquiry

```text
Hi, I can help with that. I build reusable AI agent skill packs that include
clear operating instructions, safety boundaries, test cases, and delivery
templates.

If you share the workflow goal, platform, and a redacted example, I can suggest
whether a 1-pack, 3-pack, or multi-pack bundle is the best fit.
```

## Reply When Buyer Wants "An AI Agent"

```text
I can help define the agent workflow first and package it into reusable skill
instructions. This usually works better than starting with a vague "agent"
because the final deliverable becomes easier to test and reuse.

I would start by mapping:
- What the agent receives
- What it decides
- What it outputs
- What it must refuse or escalate
- How we test whether it works
```

## Reply When Buyer Has Existing Prompts

```text
Yes, I can turn existing prompts into a cleaner skill pack. Please send only
redacted material. I will organize it into:

- README summary
- SKILL operating instructions
- SAFETY boundaries
- TEST_CASES
- DELIVERY_TEMPLATE

If the prompt is small, this is usually a Starter package. If it includes
multiple steps or roles, it may fit Standard.
```

## Reply When Buyer Wants OpenClaw / Skill Community Compatibility

```text
Yes, I can design the pack in a portable format so it is easier to adapt for
OpenClaw-style skill systems, Codex-style workflows, Claude-style skills, or a
custom internal agent runner.

I will keep the deliverable plain-text and repo-friendly unless you need a
specific runtime format.
```

## Reply When Buyer Asks For Private Access

```text
For safety, I cannot log into your accounts or handle secrets. You can send a
redacted screenshot, a sanitized sample input/output, or a short description of
the workflow. That is enough for me to design the skill pack structure and test
cases.
```

## Reply When Buyer Asks For Guaranteed Results

```text
I can make the workflow clearer, safer, and easier to test, but I cannot
guarantee platform approval, search ranking, revenue, or model behavior across
all future model versions.

What I can deliver is a structured skill pack with boundaries, test cases, and a
handoff template so you can validate it against your workflow.
```

## Minimum Buyer Requirements

Do not start until the buyer provides:

- Workflow goal
- Target platform or runner
- Redacted sample input
- Expected output format
- Safety boundaries or "must not do" list
- Existing prompt/docs/logs if they want adaptation

## Safe Delivery Message

```text
Delivery is ready. I included the skill pack files, safety boundaries, test
cases, and a delivery template. Please test the pack with your redacted sample
first, then adapt it to your live workflow only after your own review.

I did not include or request secrets, private credentials, payment details, KYC
material, or private customer data.
```

## Stop Conditions

Stop and ask the owner before:

- Accepting private repository access.
- Handling buyer account credentials.
- Opening suspicious external links.
- Making claims about guaranteed revenue or rankings.
- Publishing anything under the buyer's account.
- Touching payment, KYC, tax, wallet, subscription, or payout settings.
