# Agent Skill Pack Video Shot List

This is a record-ready shot list for a 45-second demo. Use it with
`docs/AGENT_SKILL_PACK_DEMO_AND_COMMUNITY_POST.md`.

## Recording Setup

- Record only the public repository and local public assets.
- Browser tabs to prepare:
  1. `https://github.com/butianhua4/agent-skill-packs`
  2. `assets/fiverr-gallery-contact-sheet/fiverr-gallery-contact-sheet.png`
  3. `docs/examples/ai-workflow-log-debug-filled-sample.md`
  4. `packs/ai-workflow-log-debug-skill/SAFETY.md`
  5. `docs/AGENT_SKILL_PACK_CUSTOM_OFFER_SHORT.md`
- Do not show Fiverr inbox, orders, buyer names, payment pages, KYC pages,
  wallets, OAuth screens, cookies, tokens, private repos, or personal files.

## 45-Second Shot List

| Time | Screen | Action | On-screen text | Voiceover |
| --- | --- | --- | --- | --- |
| 0-5s | GitHub README | Show repo title and first screen. | Reusable AI agent skill packs, not loose prompts. | Most agent workflows fail because the instructions are not packaged, tested, or bounded. |
| 5-9s | `packs/` folder | Scroll through pack folders slowly. | 12 validated public packs. | This repository turns repeatable AI work into structured skill packs. |
| 9-13s | One pack folder | Open a pack and show file names. | README / SKILL / SAFETY / TEST_CASES / DELIVERY_TEMPLATE | Each pack has instructions, safety rules, tests, and a delivery template. |
| 13-19s | Contact sheet PNG | Show all five gallery cards. | Marketplace-ready proof assets. | The repo includes gallery cards for explaining the offer quickly. |
| 19-24s | Filled sample output | Show a sample report heading and output sections. | Filled delivery samples included. | Buyers can inspect the delivery format before ordering. |
| 24-30s | `SAFETY.md` | Highlight boundaries and refusal rules. | Redacted inputs only. No secrets. | The workflow is built around safe, redacted inputs and clear boundaries. |
| 30-34s | Safety gallery card | Show included/not-included sections. | No payment, KYC, wallet, or account-access work. | It avoids credentials, payment settings, KYC, wallets, and unsafe account access. |
| 34-40s | Custom offer doc | Show Starter / Standard / Premium sections. | Start with 1 pack, 3 packs, or a reusable workflow system. | I can scope a small pack first, then expand into a reusable system. |
| 40-45s | README or package ladder card | End on repo URL or package ladder. | Send a workflow idea, failed logs, or an existing prompt. | Send the workflow you want to package, and I will scope the safest next step. |

## Final QA Before Posting

- Run `npm run preflight`.
- Confirm no private tabs are visible.
- Confirm no customer names or messages are visible.
- Confirm no tokens, cookies, account settings, payment, KYC, wallet, or tax
  screens appear.
- Keep claims factual: do not promise revenue, ranking, platform approval, or
  guaranteed model behavior.
