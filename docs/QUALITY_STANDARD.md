# Pack Quality Standard

Each pack must be useful as a public asset, a client delivery aid, and a reusable
agent instruction set.

## Required Properties

- Clear customer problem and outcome.
- Explicit inputs and expected outputs.
- Safe boundaries for payments, credentials, private data, external links, and
  high-permission actions.
- Repeatable workflow with checkpoints.
- Test cases that prove the pack can be used without private customer data.
- Delivery template that can become a Fiverr or GitHub portfolio artifact.

## Import Rules

- User-owned OpenClaw packs can be imported after audit.
- Archived, broken, backup, and generated-log files should not be imported by
  default.
- Claude Code backup material is architecture-only reference. Do not copy source,
  file names that imply proprietary internals, or implementation logic.
- Prefer small packs that solve one commercial problem over huge all-in-one agents.

## Commercial Fit

Priority goes to packs that can sell as:

- AI workflow debugging
- Agent log review
- CI risk gate setup
- Automation script repair
- Agent/skill pack cleanup
- Client delivery report generation
