# Agent Skill Pack Upload Checklist

Use this checklist when publishing the 7-pack bundle on Fiverr, GitHub, OpenClaw
community pages, or domestic service platforms.

## Publishing Goal

Position the bundle as a practical, reusable AI agent operations kit:

- For buyers who already use AI agents but need better prompts, trigger rules,
  logging, error handling, progress updates, and delivery templates.
- For teams that want a safer agent workflow before adding more automation.
- For founders or operators who need a small, fast implementation rather than a
  large custom platform.

## Buyer Requirements

Ask these questions before starting:

1. What workflow should the agent or skill pack support?
2. Which platform will run it? Examples: OpenClaw, Codex, Claude-style skills,
   custom agent, n8n, Zapier, Make, GitHub Actions, or other.
3. What should the final output look like?
4. Do you have a sample input and expected output? Redacted examples are best.
5. What should the agent never do?
6. Are there existing prompts, logs, or docs I should adapt?

## User Confirmation Gates

The user must personally handle or confirm:

- Publishing a paid listing.

- Uploading personal ID or identity material.
- Sharing private client repositories or credentials.
- Accepting a custom order above the listed scope.

## Pre-Publish QA

Run:

```bash
npm run preflight
```

Check:

- All packs pass structure validation.
- No unsafe promises are added to the listing.
- Gallery assets match the service being sold.
- Package scope is small enough to deliver reliably.
- Buyer requirements ask for redacted examples instead of secrets.
