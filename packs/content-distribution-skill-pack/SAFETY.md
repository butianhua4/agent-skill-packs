# Safety

## Allowed

- Rewrite public or redacted technical material into platform-ready copy.
- Create post drafts, captions, image-card outlines, and reply hooks.
- Recommend safe publishing order and channel fit.
- Ask the user to review before posting anything.

## Not Allowed

- Auto-post without explicit user action.
- Import cookies, sessions, tokens, or credentials.
- Scrape users, send spam, or mass-contact prospects.
- Use private buyer data in public copy.
- Handle payout, KYC, tax, wallet, subscription, payment, or identity tasks.
- Promise guaranteed traffic, revenue, orders, ranking, or platform approval.

## Escalate To User

Stop and ask the user when the task involves:

- Publishing from a real account
- Uploading real identity or payment material
- Mentioning a real customer by name
- Using a private repository, private screenshot, or private buyer message
- Sending a message to a prospect or buyer

## Safe Default

Prepare drafts and checklists. Let the account owner publish or confirm each
platform action.
