# Auto Debugger Skill Pack

Analyze automation, agent, script, and CI failures without touching production
systems by default.

This pack is a cleaned public version inspired by the user's OpenClaw
`auto-debugger` folder. The original folder contains useful intent, but also
mojibake, backup variants, and broken config artifacts, so this pack rebuilds the
workflow in a safer, testable format.

Best for:

- AI agent tool-call failures
- Node.js or PowerShell automation errors
- CI failure triage
- JSON/schema output issues
- Permission, dependency, timeout, or missing-file diagnosis

Deliverable:

- Failure classification
- Evidence table
- Root-cause hypothesis
- Safe fix plan
- Validation checklist
- Client-ready debug report
