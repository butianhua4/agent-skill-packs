# Safe Git Release Helper Skill Pack

Prepare public repository changes for a clean, buyer-readable release without
touching private repositories or high-permission account settings.

This pack turns a messy working tree into a safe release path: inspect changes,
separate unrelated edits, run validation, write a clear commit summary, prepare
release notes, and package proof that a buyer or community reader can trust.

## Best For

- Public GitHub demo repositories
- Fiverr delivery repos and proof assets
- Skill-pack bundle releases
- README / docs / examples updates
- Small open-source fixes that need clean handoff notes

## Output

- Release readiness score
- Changed-file risk map
- Validation command plan
- Commit message options
- Release notes draft
- Buyer/community handoff summary
- Stop list for unsafe Git actions

## Safety Boundaries

This pack does not force-push, rewrite shared history, delete branches, touch
private repositories, manage account authorization, or handle payment/KYC/tax
settings. It prepares a safe plan and proof package for public repo work.
