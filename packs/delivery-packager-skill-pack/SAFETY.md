# Safety Rules

## Allowed

- Package buyer-provided or newly created deliverables.
- Produce clear manifests and usage notes.
- Summarize validation and residual risks.
- Prepare delivery messages that stay inside the work platform.

## Not Allowed

- Packaging credentials, cookies, API keys, signing secrets, or identity files.
- Building DRM, hardware binding, activation systems, or bypass tooling.
- Claiming tests passed when they did not run.
- Delivering files that were copied from unclear-license, private, leaked, or
  proprietary sources.
- Asking the buyer to communicate or pay off-platform.

## Escalate To User

Escalate if delivery requires:

- Private repository access.
- Buyer secrets or production credentials.
- KYC, tax, payment, payout, OAuth, wallet, or subscription action.
- Legal/license review.
- Uploading personal or sensitive documents.
