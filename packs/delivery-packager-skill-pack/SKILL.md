# Delivery Packager Skill

## Purpose

Create a safe, buyer-readable delivery package for completed AI automation,
agent workflow, debugging, documentation, or skill-pack work.

## Inputs

- Buyer goal and order scope.
- Completed files or deliverables.
- Validation commands and results.
- Setup steps or usage instructions.
- Known limitations and excluded work.
- Follow-up recommendations.

## Workflow

1. Confirm the delivery scope and exclude anything not completed.
2. Build a manifest that lists each deliverable and its purpose.
3. Add setup or usage instructions in plain language.
4. Add validation evidence with command names, dates, and results.
5. Add a limitations section that avoids overpromising.
6. Add an acceptance checklist the buyer can use to review the work.
7. Prepare a short platform-safe delivery message.

## Output

The final package should include:

- `delivery-manifest.md`
- `setup-or-usage.md`
- `validation-summary.md`
- `known-limitations.md`
- `acceptance-checklist.md`
- `buyer-delivery-message.md`

## Boundaries

- Do not package secrets, tokens, cookies, private keys, or identity documents.
- Do not create activation locks, hardware binding, or license enforcement.
- Do not include unverified claims.
- Do not include suspicious buyer links.
- Do not handle payments, payout, tax, KYC, OAuth, subscription, or wallet tasks.
