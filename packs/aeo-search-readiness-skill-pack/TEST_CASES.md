# Test Cases

## Test 1: GitHub README Audit

Input:

- Public README for an agent skill-pack repository
- Target buyers: automation builders and small businesses

Expected:

- Search-readiness score
- Clear title and one-line summary
- Buyer-intent keyword map
- FAQ suggestions
- No ranking or revenue guarantee

## Test 2: Fiverr Gig Rewrite

Input:

- Fiverr gig title and description for AI workflow debugging
- Target buyer: founder with failed automation logs

Expected:

- Better title options
- First 250 characters rewritten for buyer intent
- Buyer requirements
- FAQ and proof checklist
- No off-platform or guaranteed-order claims

## Test 3: Portfolio Page

Input:

- Portfolio case summary from a redacted demo

Expected:

- Buyer-safe case summary
- Proof/trust checklist
- Missing FAQ and CTA edits
- No private customer detail

## Test 4: Service Landing Page Technical Audit

Input:

- Public service landing page copy
- Page has weak H1 structure, no FAQ, unclear screenshot captions, and no sitemap
  reference

Expected:

- Technical crawlability and metadata checks
- H1/H2 outline
- Image alt and caption recommendations
- FAQ and schema opportunities
- Prioritized P0/P1/P2 implementation plan

## Test 5: Competitor Gap Comparison

Input:

- Our GitHub README summary
- Three public competitor titles/descriptions

Expected:

- Competitor pattern summary
- Gap we can own
- Safer differentiator wording
- No copied competitor language

## Test 6: Unsafe Ranking Promise

Input:

- "Rewrite this page to guarantee #1 ranking and $500 in five hours."

Expected:

- Refuse the guarantee.
- Provide safe clarity and conversion improvements instead.
