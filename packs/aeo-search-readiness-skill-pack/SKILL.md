# AEO Search Readiness Skill

## Purpose

Audit and improve public-facing technical service assets so buyers, search
engines, and answer engines can understand them. Use this for GitHub READMEs,
Fiverr gigs, portfolio pages, productized service pages, marketplace listings,
and community posts.

This is a full search-readiness and answer-readiness workflow, not only a copy
rewrite workflow. It covers buyer intent, technical crawlability, on-page
metadata, heading structure, structured data opportunities, image/media signals,
FAQ coverage, proof signals, competitor gaps, and conversion clarity.

## Inputs

Accept one or more of:

- Public README, landing page, gig description, FAQ, or listing copy
- Service title, package table, tags, and buyer requirements
- Portfolio proof, demo link, release note, or sample report summary
- Target buyer, target platform, and search intent
- Competitor titles or public descriptions

Reject or ask for redaction when inputs contain:

- Passwords, tokens, cookies, secret values, payout details, KYC material, or ID
  documents
- Private buyer names, private repositories, private screenshots, or non-public
  customer data
- Requests to promise rankings, traffic, revenue, platform approval, or buyer
  volume

## Workflow

1. Identify the asset type:
   - GitHub README
   - Fiverr gig
   - Portfolio page
   - Service landing page
   - Community post
2. Extract the current search promise:
   - Who it helps
   - Pain solved
   - Deliverable
   - Proof
   - Safety boundary
   - Call to action
3. Run the full readiness audit:
   - Buyer intent: target buyer, pain phrases, deliverable phrases, urgency,
     exclusions, and decision objections
   - Technical crawlability: public URL clarity, indexable page assumptions,
     robots/sitemap references, canonical hints, broken navigation hints, and
     whether the asset is accessible without login
   - On-page metadata: title, meta description, first paragraph, H1/H2/H3
     hierarchy, URL slug, social preview title, and excerpt quality
   - AEO structure: direct answer blocks, FAQ questions, entity definitions,
     comparison snippets, step-by-step sections, and schema opportunities
   - Content quality: specificity, evidence, examples, before/after proof,
     buyer requirements, delivery boundaries, and skim readability
   - Media readiness: image alt text, screenshot labels, demo captions, file
     names, compression notes, and whether media proves the service outcome
   - Internal journey: next step, call to action, relevant links, package fit,
     and buyer handoff requirements
   - Competitor gap: compare public titles/descriptions when supplied and find
     missing buyer language, proof, service scope, and trust signals
4. Score readiness from 0-100:
   - Clear buyer promise: 15
   - Buyer-intent language: 15
   - Concrete deliverable: 15
   - Technical crawlability and metadata: 15
   - AEO / FAQ / structured answer coverage: 15
   - Proof, trust, and conversion path: 15
   - Safe boundaries: 10
5. Find gaps:
   - Missing buyer words
   - Vague outcome
   - Weak proof
   - Overbroad promise
   - Missing FAQ
   - Missing requirements
   - Missing title or metadata
   - Weak heading structure
   - No structured answer section
   - Missing image alt text or media captions
   - Missing public demo, sample report, or case proof
   - Missing canonical/sitemap/robots notes for owned web pages
6. Rewrite or specify the most important blocks:
   - Title
   - One-line summary
   - Description opening
   - Meta title and meta description for owned pages
   - H1/H2 outline
   - FAQ
   - Direct answer snippets
   - Schema recommendation
   - Buyer requirements
   - Tags / keywords
7. Produce a prioritized next-edit checklist:
   - P0: fixes that block understanding or trust
   - P1: fixes that improve discoverability and conversion
   - P2: polish and measurement improvements

## Output

Return a concise Markdown report:

- Search-readiness score
- Executive diagnosis
- Buyer-intent keyword map
- Technical crawlability and metadata checks
- AEO / structured answer recommendations
- Rewritten title and summary
- Improved description opening
- Recommended heading outline
- FAQ block
- Image/media proof checklist
- Competitor gap summary when competitor examples are supplied
- Proof/trust checklist
- Safety edits
- Prioritized next 30-minute action

## Boundaries

- Do not promise rankings, traffic, revenue, virality, orders, or platform
  approval.
- Do not use private buyer data or confidential screenshots as public proof.
- Do not recommend black-hat SEO, spam, scraped reviews, fake testimonials, or
  keyword stuffing.
- Do not handle payout, wallet, KYC, tax, payment, account recovery, or identity
  tasks.
- Prefer clear buyer intent and verifiable proof over hype.
