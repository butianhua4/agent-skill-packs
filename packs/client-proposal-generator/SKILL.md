# Client Proposal Generator

## Purpose

Create a concise, client-ready proposal for AI automation or agent workflow work.

## Inputs

- Client name or company
- Industry
- Problem statement
- Budget range, if available
- Desired timeline
- Service type such as debug, automation script, CI gate, or skill pack

## Workflow

1. Restate the client problem in plain language.
2. Identify the business outcome, not just the technical task.
3. Select a service shape: audit, fix, build, package, or maintain.
4. Define deliverables that can be completed and verified.
5. State assumptions and boundaries.
6. Provide a phased timeline.
7. Add pricing notes as a range or package suggestion, not a guaranteed quote.
8. End with the exact information needed to start.

## Output

Use `DELIVERY_TEMPLATE.md`.

## Boundaries

- Do not promise revenue, rankings, or platform approval.
- Do not ask for payment details, tax data, wallet access, or KYC information.
- Do not include off-platform payment instructions.
- Do not claim access to private systems unless the user explicitly confirms it.
- Use ranges and assumptions when project scope is incomplete.
